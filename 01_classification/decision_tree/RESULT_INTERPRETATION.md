# Decision Tree Classification — Result Interpretation

## 1. Purpose of This Document

This document explains how to interpret the results generated by the Decision Tree Classification model.

The model was trained using the Breast Cancer Wisconsin dataset to classify observations as:

- `0` — Malignant
- `1` — Benign

The purpose of this interpretation is not only to report model performance, but also to explain what the metrics, visualizations, and model structure mean from both technical and practical perspectives.

---

## 2. Model Overview

A Decision Tree classifier makes predictions by dividing the dataset through a sequence of feature-based rules.

At each node, the algorithm selects a feature and threshold that best separates the classes.

For example, a rule may look like:

```text
worst radius <= 16.8

Depending on whether the condition is true or false, the observation moves to the left or right branch.

This process continues until the observation reaches a leaf node, where the final class prediction is made.

The model was configured with:

criterion = "gini"
max_depth = 5
min_samples_split = 10
min_samples_leaf = 5
random_state = 42

These settings help control tree complexity and reduce overfitting.

3. Dataset Class Meaning

The target labels in the scikit-learn Breast Cancer Wisconsin dataset are:

Class Value	Class Name
0	Malignant
1	Benign

This class order is important because the positive class used by many scikit-learn metrics is class 1, which represents benign cases.

Therefore:

Precision refers to benign predictions unless otherwise specified.
Recall refers to correctly identified benign cases unless otherwise specified.
ROC-AUC uses the benign class as the positive class in the current implementation.

For medical interpretation, class-specific metrics should also be reviewed because correctly detecting malignant cases may be more important than overall accuracy.

4. Accuracy

Accuracy measures the proportion of all predictions that were correct.

The formula is:

Accuracy = Correct Predictions / Total Predictions

Example interpretation:

An accuracy of 0.94 means that the model correctly classified approximately 94% of the observations in the test dataset.

Accuracy provides a general overview of performance.

However, accuracy alone can be misleading when:

one class is much larger than the other;
different errors have different consequences;
false negatives are more serious than false positives.

In a medical classification context, incorrectly predicting a malignant case as benign may be more serious than incorrectly predicting a benign case as malignant.

Therefore, accuracy should always be interpreted together with precision, recall, F1-score, ROC-AUC, and the confusion matrix.

5. Precision

Precision measures how many observations predicted as the positive class were actually positive.

The formula is:

Precision = True Positives / (True Positives + False Positives)

Because class 1 represents benign cases, precision in this implementation answers the following question:

Of all observations predicted as benign, how many were actually benign?

A high precision score means that the model makes relatively few false positive predictions for the benign class.

Example interpretation:

A precision of 0.96 means that approximately 96% of the observations predicted as benign were truly benign.

In this context, a false positive for the benign class means that a malignant case was incorrectly predicted as benign.

This type of error can be important in medical decision-support applications.

6. Recall

Recall measures how many actual positive observations were correctly identified.

The formula is:

Recall = True Positives / (True Positives + False Negatives)

Because benign is the positive class in this implementation, recall answers:

Of all truly benign observations, how many did the model correctly identify as benign?

Example interpretation:

A recall of 0.97 means that the model correctly identified approximately 97% of all benign observations.

High recall for the benign class means the model misses very few benign observations.

However, it does not directly show how well malignant cases are detected.

For medical use, recall for the malignant class should also be inspected through the classification report or calculated separately.

7. F1-Score

The F1-score combines precision and recall into a single metric.

The formula is:

F1 = 2 × (Precision × Recall) / (Precision + Recall)

The F1-score is useful when both false positives and false negatives matter.

Example interpretation:

An F1-score of 0.95 indicates that the model achieves a strong balance between precision and recall for the benign class.

A high F1-score means that the model is performing well without sacrificing either precision or recall too heavily.

However, the F1-score does not consider true negatives directly.

8. ROC-AUC

ROC-AUC stands for Receiver Operating Characteristic — Area Under the Curve.

It measures how well the model separates the two classes across many possible classification thresholds.

General interpretation:

ROC-AUC Score	Interpretation
0.50	No better than random
0.60–0.70	Weak
0.70–0.80	Acceptable
0.80–0.90	Strong
Above 0.90	Excellent

Example interpretation:

A ROC-AUC score of 0.95 indicates excellent separation between benign and malignant observations.

A high ROC-AUC means the model generally assigns higher probabilities to the positive class for positive observations than for negative observations.

ROC-AUC is useful because it evaluates ranking ability across thresholds rather than at only the default threshold of 0.50.

However, ROC-AUC should not be treated as sufficient on its own.

For imbalanced or high-risk applications, precision-recall analysis may also be useful.

9. Confusion Matrix

The confusion matrix shows the number of correct and incorrect predictions by class.

With the current class order:

Actual Class	Predicted Class	Interpretation
Malignant	Malignant	Correct malignant classification
Malignant	Benign	Malignant case incorrectly classified as benign
Benign	Malignant	Benign case incorrectly classified as malignant
Benign	Benign	Correct benign classification

The confusion matrix can be represented as:

[[True Malignant, False Benign],
 [False Malignant, True Benign]]

More traditionally:

[[True Negatives, False Positives],
 [False Negatives, True Positives]]

Because class 0 is malignant and class 1 is benign:

True negatives are correctly classified malignant cases.
False positives are malignant cases incorrectly classified as benign.
False negatives are benign cases incorrectly classified as malignant.
True positives are correctly classified benign cases.

This can be confusing in medical contexts because the medically important condition is encoded as class 0.

For a production medical system, malignant cases could instead be encoded as the positive class to make medical-risk interpretation more intuitive.

10. Medical Risk Interpretation

The two main classification errors have different practical meanings.

Malignant predicted as benign

This is potentially the more serious error.

A malignant observation incorrectly predicted as benign could delay further investigation or treatment.

From a medical risk perspective, this error should be minimized.

Benign predicted as malignant

This may lead to additional tests, anxiety, or unnecessary follow-up procedures.

Although still undesirable, it is usually less dangerous than missing a malignant case.

Therefore, model threshold selection should consider the cost of each error rather than relying only on default classification settings.

11. Classification Report

The classification report provides precision, recall, and F1-score for each class.

It typically includes:

precision;
recall;
F1-score;
support;
macro average;
weighted average.
Support

Support is the number of actual observations belonging to each class.

This helps determine whether one class is represented more heavily than the other.

Macro Average

The macro average calculates the metric separately for each class and then takes the simple average.

Each class receives equal importance.

This is useful when both classes should be treated equally.

Weighted Average

The weighted average calculates the metric for each class and weights it according to the number of observations in that class.

This reflects overall dataset composition but may hide weaker performance on the smaller class.

For medical applications, class-specific recall should be reviewed carefully rather than relying only on weighted averages.

12. Feature Importance

The Decision Tree provides a feature-importance score for each input variable.

Feature importance indicates how much each feature contributed to reducing impurity across the tree.

Features with higher values had a greater influence on the model's decision rules.

Example interpretation:

If worst radius has the highest importance, it means that this feature played a major role in separating benign and malignant observations.

Important limitations:

Feature importance does not prove causality.
Correlated features may divide importance between them.
Importance values can change if the training data changes.
A low-importance feature is not necessarily medically irrelevant.
Decision Tree importance can be biased toward features that allow many possible split points.

For stronger interpretability, SHAP values or permutation importance could be added in a later version.

13. Decision Tree Structure

The decision tree visualization shows the exact sequence of decisions used by the model.

Each node commonly contains:

14. Gini Impurity

Gini impurity measures how mixed the classes are within a node.

A Gini score of:

0.0 means the node contains observations from only one class;
a higher value means the node contains a mixture of classes.

For binary classification, Gini impurity reaches its maximum near 0.5 when the two classes are evenly mixed.

The Decision Tree chooses splits that reduce Gini impurity as much as possible.

Example interpretation:

A child node with lower Gini impurity is more class-homogeneous than its parent node.

15. Tree Depth

Tree depth refers to the number of levels between the root node and the deepest leaf.

The model uses:

max_depth = 5

This means the tree cannot grow beyond five decision levels.

A shallow tree:

is easier to interpret;
is less likely to overfit;
may underfit complex relationships.

A deep tree:

can learn complex patterns;
may achieve high training accuracy;
is more likely to overfit;
is harder to interpret.

The selected depth is a compromise between model complexity and generalization.

16. Number of Leaves

Leaf nodes contain the final class predictions.

A larger number of leaves allows the tree to represent more detailed patterns.

However, too many leaves can make the model overly specialized to the training dataset.

The parameters:

min_samples_split = 10
min_samples_leaf = 5

help prevent the creation of very small and unstable branches.

17. Overfitting Assessment

Decision Trees are highly flexible and can overfit easily.

Overfitting occurs when the model learns noise or dataset-specific patterns rather than general relationships.

Signs of overfitting include:

extremely high training accuracy;
substantially lower test accuracy;
very deep tree structure;
many small leaf nodes;
unstable predictions when the data changes slightly.

The current model controls overfitting using:

max_depth = 5
min_samples_split = 10
min_samples_leaf = 5

A future improvement would compare:

training accuracy;
validation accuracy;
test accuracy;
performance at different tree depths.

This could be visualized using a validation curve.

18. Underfitting Assessment

Underfitting occurs when the model is too simple to capture meaningful patterns.

Possible signs include:

low training accuracy;
low test accuracy;
very shallow tree;
overly restrictive pruning settings.

If underfitting occurs, possible improvements include:

increasing max_depth;
reducing min_samples_split;
reducing min_samples_leaf;
testing the entropy criterion;
engineering more informative features.

Any increase in model complexity should be evaluated using cross-validation.

19. Probability Predictions

The model produces probability estimates using:

model.predict_proba()

For each observation, the model returns probabilities for both classes.

Example:

Malignant probability: 0.80
Benign probability: 0.20

The predicted class is usually the class with the highest probability.

However, these probabilities should be interpreted carefully because Decision Tree probabilities are based on class proportions in terminal leaves.

They may not be well calibrated.

Probability calibration methods such as:

Platt scaling;
isotonic regression;
calibration curves;

could be considered for future improvements.

20. Threshold Interpretation

The default classification threshold is usually 0.50.

This means:

If probability of class 1 >= 0.50:
    predict benign
Else:
    predict malignant

For medical risk applications, the threshold could be adjusted.

For example, a stricter threshold for predicting benign could reduce the chance of incorrectly classifying malignant cases as benign.

Threshold selection should be based on:

cost of false positives;
cost of false negatives;
operational objectives;
clinical requirements;
sensitivity and specificity targets.

23. Limitations of the Model

The main limitations are:

prone to overfitting;
sensitive to small changes in data;
greedy split selection may not find the global optimum;
feature importance can be biased;
probability outputs may be poorly calibrated;
a single tree may be less accurate than ensemble methods;
performance can depend strongly on hyperparameters.

Random Forest and Gradient Boosting often improve predictive performance by combining multiple trees.

24. Comparison With Logistic Regression

Decision Tree Classification and Logistic Regression solve similar classification problems differently.

Aspect	Decision Tree	Logistic Regression
Relationship type	Nonlinear	Mainly linear decision boundary
Interpretability	Rule-based tree	Coefficient-based
Feature scaling	Usually not required	Often recommended
Feature interactions	Learned automatically	Must often be specified
Overfitting risk	Relatively high	Usually lower
Stability	Can be unstable	Generally more stable
Probability calibration	Often weaker	Often better
Complex patterns	Stronger	More limited

The Decision Tree is useful when nonlinear rules and explicit decision paths are important.

Logistic Regression is useful when a stable, interpretable baseline with coefficient-based explanations is preferred.
